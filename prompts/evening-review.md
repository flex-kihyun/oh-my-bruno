# Evening Review Prompt

이 파일은 `evening-review` 스케줄 태스크의 실행 지시사항이다.
매 평일 저녁 자동 실행되어 오늘의 회고를 기존 Daily 페이지에 append한다.

---

## 실행 순서

### 0. 설정 로드

먼저 `config.md` 파일을 읽어서 Notion DB ID, 페이지 ID 등 설정값을 확인한다.

### 1. 오늘의 Daily 페이지 찾기

Notion에서 오늘의 Daily 페이지를 찾는다:
- `notion-search`로 오늘 날짜 + "Daily Check-in" 검색
- DB: config의 `DAILY_DB` 값 사용
- 카테고리: config의 `DAILY_CATEGORY`로 필터
- 찾은 페이지를 `notion-fetch`로 전체 콘텐츠 로드

**페이지를 못 찾은 경우**: 오늘 아침 브리핑이 생성되지 않았을 수 있음. 이 경우 새 페이지를 생성하되, 아침 브리핑 없이 저녁 회고만 포함.

### 2. 사용자 피드백 읽기

Daily 페이지의 "💬 오늘의 피드백" 섹션을 파싱:
- "오늘 브리핑에서 가장 유용했던 것" → 기록
- "불필요했던 것" → 기록
- "추가로 알고 싶은 것" → 기록
- "기타 메모" → 기록

각 항목이 비어있으면 해당 항목은 스킵.

### 3. 피드백 반영

피드백이 있는 경우:
1. **저녁 회고에 "📋 피드백 반영" 섹션 포함**: 사용자 피드백 요약 + 어떻게 반영할지 설명
2. **Bruno Config 업데이트**: 
   - "불필요했던 것"이 있으면 → Briefing Preferences 페이지(config의 `BRIEFING_PREFS_PAGE`)의 "피드백 히스토리" 섹션에 추가
   - "추가로 알고 싶은 것"이 있으면 → Briefing Preferences에 다음 브리핑 추가 수집 항목으로 기록
   - `notion-update-page`로 Briefing Preferences 페이지 업데이트

### 4. 데이터 수집

접근 가능한 **모든 MCP 도구**를 활용:

#### Linear
- 아침 브리핑에 있던 이슈들의 현재 상태 확인
- `list_issues`로 오늘 상태가 변경된 이슈 (Done, Closed로 이동한 것들)
- 새로 생성된 이슈 확인

#### Google Calendar
- 오늘 일정 중 실제로 진행된 미팅 목록
- 총 미팅 시간 계산

#### Slack
- 오늘 내가 보낸/받은 주요 메시지 요약
- 해결된 스레드, 새로운 요청 등

#### Git / GitHub
- 오늘 커밋 수, PR 상태 변화
- 머지된 PR, 새로 열린 PR

#### 아침 브리핑 대비
- 아침에 "오늘의 할 일"로 나열된 것 vs 실제 완료한 것 비교
- 블로커가 해결되었는지 확인

### 5. 세션 인사이트 자동 수집 (User Profile 진화)

오늘 수집한 데이터에서 사용자의 **결정 패턴, 선호도, 성향**을 추출하여 User Profile을 자동 업데이트한다.

#### 수집 대상

**Linear에서 읽는 패턴**:
- 자주 사용하는 라벨/카테고리 → "선호 작업 유형" 기록
- 이슈 처리 순서 → "우선순위 결정 패턴" (고객 이슈 우선? 프로젝트 우선?)
- 평균 이슈 처리 시간 → "작업 속도 패턴"

**Calendar에서 읽는 패턴**:
- 미팅 빈도 변화 → "미팅 부하 트렌드"
- 자주 만나는 사람 → "핵심 협업자"
- 미팅 없는 시간대 → "실제 포커스 타임"

**Slack에서 읽는 패턴**:
- 자주 활동하는 채널 → "관심 영역"
- 응답 속도 패턴 → "커뮤니케이션 스타일"
- 자주 언급하는 주제 → "현재 관심사"

**Git에서 읽는 패턴**:
- 커밋 시간대 → "생산적인 시간대"
- 자주 수정하는 파일/모듈 → "전문 영역"

#### 업데이트 방식

1. 추출한 인사이트를 User Profile 페이지(config의 `USER_PROFILE_PAGE`)에 반영
2. **기존 내용을 덮어쓰지 않고**, "## 자동 수집 인사이트" 섹션을 업데이트
3. 날짜와 함께 기록하여 변화 추이를 볼 수 있게
4. 명확한 패턴만 기록 (1회성 행동은 무시, 3일 이상 반복된 패턴만 기록)

#### 저녁 회고에 포함

수집한 인사이트 중 오늘 새로 발견된 것이 있으면 저녁 회고에 "🔍 오늘 발견한 패턴" 섹션으로 간단히 표시:
```markdown
## 🔍 오늘 발견한 패턴
- 오후 2~5시에 커밋이 집중됨 → 포커스 타임 확인
- CI-* 이슈 대응에 평균 1.5시간 소요
```

### 6. Notion Daily 페이지에 append

`notion-update-page`의 `update_content` 커맨드로 기존 페이지 끝에 저녁 회고 섹션 추가.

**콘텐츠 구조** (`templates/daily-page.md`의 "저녁 회고 추가 구조" 참조):

```markdown
---

## 🌙 Buona sera, 형!

## 📋 피드백 반영
> 형이 오늘 남긴 피드백 기반으로 정리했어:
- {피드백 요약}
- {반영 내용}

## ✅ 오늘 완료한 것
- {완료된 이슈/작업 목록}

## 🔄 진행 중 (내일 계속)
- {진행 중인 이슈들}

## 🚧 블로커
- {미해결 블로커}

## 📊 오늘의 숫자
- Linear: N개 완료, N개 진행 중, N개 신규
- 미팅: N개 참석 (총 N시간)
- 커밋: N개
- PR: N개 열림, N개 머지

## 🔍 오늘 발견한 패턴
- {새로 발견된 사용자 패턴/인사이트}
- {User Profile에 업데이트한 내용}
```

피드백이 없으면 "📋 피드백 반영" 섹션은 생략.

### 7. 작성 스타일

`style-guide.md`를 읽고 Bruno 페르소나에 맞게 작성한다.
저녁 인사는 "🌙 Buona sera, 형!"으로 시작.

### 8. 에러 처리

- Daily 페이지를 못 찾으면 새 페이지 생성 (저녁 회고만 포함)
- 특정 도구 실패 시 나머지 데이터로 회고 작성
- append 실패 시 재시도 1회, 그래도 실패하면 새 페이지로 생성
