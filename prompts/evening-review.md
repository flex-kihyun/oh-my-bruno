# Evening Review Prompt

이 파일은 `evening-review` 스케줄 태스크의 실행 지시사항이다.
매 평일 저녁 자동 실행되어 오늘의 회고를 **별도 Notion 페이지**로 생성한다.

---

## 실행 순서

### 0. 설정 로드

먼저 `config.md` 파일을 읽어서 Notion DB ID, 페이지 ID, **TIMEZONE** 등 설정값을 확인한다.

**타임존 처리**:
- 리모트 트리거는 UTC로 실행되지만, "오늘"의 기준은 `TIMEZONE`(Asia/Seoul, UTC+09:00)이다.
- 현재 시각을 `TIMEZONE`으로 변환한 뒤 날짜(`YYYY-MM-DD`)와 요일을 계산한다.
- Calendar/Linear/Slack/Git 조회 시 시간 범위는 `TIMEZONE` 기준 자정을 ISO 8601(`2026-04-17T00:00:00+09:00` 형태)로 변환.

### 1. 오늘의 Daily 페이지 찾기

Notion에서 오늘(`TIMEZONE` 기준)의 Daily 페이지를 찾는다:
- `notion-search`로 오늘 날짜(`YYYY-MM-DD`, `TIMEZONE` 기준) + "Daily Check-in" 검색
- DB: config의 `DAILY_DB` 값 사용
- 카테고리: config의 `DAILY_CATEGORY`로 필터
- 찾은 페이지를 `notion-fetch`로 전체 콘텐츠 로드

**페이지를 못 찾은 경우**: 오늘 아침 브리핑이 생성되지 않았을 수 있음. 이 경우 새 페이지를 생성하되, 아침 브리핑 없이 저녁 회고만 포함.

### 2. 사용자 피드백 읽기

Daily 페이지의 "💬 오늘의 피드백" 섹션을 파싱:
- "오늘 브리핑에서 가장 유용했던 것" → 기록
- "불필요했던 것" → 기록
- "추가로 알고 싶은 것" → 기록
- "기타 메모" → 기록

각 항목이 비어있으면 해당 항목은 스킵.

### 3. 피드백 반영

피드백이 있는 경우:
1. **저녁 회고에 "📋 피드백 반영" 섹션 포함**: 사용자 피드백 요약 + 어떻게 반영할지 설명
2. **Bruno Config 업데이트**: 
   - "불필요했던 것"이 있으면 → Briefing Preferences 페이지(config의 `BRIEFING_PREFS_PAGE`)의 "피드백 히스토리" 섹션에 추가
   - "추가로 알고 싶은 것"이 있으면 → Briefing Preferences에 다음 브리핑 추가 수집 항목으로 기록
   - `notion-update-page`로 Briefing Preferences 페이지 업데이트

### 4. 데이터 수집

접근 가능한 **모든 MCP 도구**를 활용:

#### Linear
- 아침 브리핑에 있던 이슈들의 현재 상태 확인
- `list_issues`로 오늘 상태가 변경된 이슈 (Done, Closed로 이동한 것들)
- 새로 생성된 이슈 확인

#### Google Calendar
- 오늘 일정 중 실제로 진행된 미팅 목록
- 총 미팅 시간 계산

#### Slack
- 오늘 내가 보낸/받은 주요 메시지 요약
- 해결된 스레드, 새로운 요청 등

#### Git / GitHub
- 오늘 커밋 수, PR 상태 변화
- 머지된 PR, 새로 열린 PR

#### 아침 브리핑 대비
- 아침에 "오늘의 할 일"로 나열된 것 vs 실제 완료한 것 비교
- 블로커가 해결되었는지 확인

#### Claude 세션 업무 추적
오늘 Claude를 통해 처리한 업무와 주요 결정사항을 수집한다.
Claude 세션의 흔적은 여러 곳에 남아있으므로 교차 확인:

1. **Git 커밋 메시지 분석**
   - 오늘 날짜 커밋 중 `Co-Authored-By: Claude` 포함된 것 필터링
   - 커밋 메시지에서 작업 내용과 결정사항 추출
   - 어떤 레포에서 어떤 변경이 있었는지 요약

2. **Linear 이슈 코멘트/업데이트**
   - 오늘 업데이트된 이슈의 코멘트에서 Claude 세션 관련 내용 확인
   - 이슈 상태 변경 히스토리 (누가 언제 변경했는지)
   - 새로 생성된 이슈나 서브이슈

3. **GitHub PR/리뷰**
   - 오늘 생성/업데이트된 PR의 description과 리뷰 코멘트
   - Claude가 작성한 PR 내용 요약
   - 리뷰에서 나온 주요 논의사항

4. **Notion 페이지 변경**
   - 오늘 생성/수정된 Notion 페이지 확인
   - 스펙 문서, 설계 문서, 기술 문서 변경사항

5. **Slack 스레드**
   - Claude 관련 논의가 포함된 스레드 확인
   - 기술적 결정이나 방향성 논의

수집한 내용을 **"🤖 오늘 Claude와 함께 한 일"** 섹션으로 정리:
- 처리한 업무 목록 (무엇을, 어떤 레포에서)
- 주요 결정사항 (왜 그렇게 결정했는지 맥락 포함)
- 생성된 산출물 (PR, 문서, 코드 등)

### 5. 세션 인사이트 자동 수집 (User Profile 진화)

오늘 수집한 데이터에서 사용자의 **결정 패턴, 선호도, 성향**을 추출하여 User Profile을 자동 업데이트한다.

#### 수집 대상

**Linear에서 읽는 패턴**:
- 자주 사용하는 라벨/카테고리 → "선호 작업 유형" 기록
- 이슈 처리 순서 → "우선순위 결정 패턴" (고객 이슈 우선? 프로젝트 우선?)
- 평균 이슈 처리 시간 → "작업 속도 패턴"

**Calendar에서 읽는 패턴**:
- 미팅 빈도 변화 → "미팅 부하 트렌드"
- 자주 만나는 사람 → "핵심 협업자"
- 미팅 없는 시간대 → "실제 포커스 타임"

**Slack에서 읽는 패턴**:
- 자주 활동하는 채널 → "관심 영역"
- 응답 속도 패턴 → "커뮤니케이션 스타일"
- 자주 언급하는 주제 → "현재 관심사"

**Git에서 읽는 패턴**:
- 커밋 시간대 → "생산적인 시간대"
- 자주 수정하는 파일/모듈 → "전문 영역"

#### 업데이트 방식

1. 추출한 인사이트를 User Profile 페이지(config의 `USER_PROFILE_PAGE`)에 반영
2. **기존 내용을 덮어쓰지 않고**, "## 자동 수집 인사이트" 섹션을 업데이트
3. 날짜와 함께 기록하여 변화 추이를 볼 수 있게
4. 명확한 패턴만 기록 (1회성 행동은 무시, 3일 이상 반복된 패턴만 기록)

#### 저녁 회고에 포함

수집한 인사이트 중 오늘 새로 발견된 것이 있으면 저녁 회고에 "🔍 오늘 발견한 패턴" 섹션으로 간단히 표시:
```markdown
## 🔍 오늘 발견한 패턴
- 오후 2~5시에 커밋이 집중됨 → 포커스 타임 확인
- CI-* 이슈 대응에 평균 1.5시간 소요
```

### 6. 별도 Notion 저녁 회고 페이지 생성

`notion-create-pages`로 **새 페이지**를 생성한다 (아침 브리핑과 별도).

**대상 DB**: config의 `DAILY_DB` 값 사용

**페이지 속성**:
- `이름`: `"Evening Review
- `카테고리`: config의 `DAILY_CATEGORY` 값 (기본: `["Daily"]`)

**아이콘**: 🌙

**콘텐츠 구조**:

```markdown
## 🌙 Buona sera, 형!

## 📋 피드백 반영
> 형이 오늘 남긴 피드백 기반으로 정리했어:
- {피드백 요약}
- {반영 내용}

## 🤖 오늘 Claude와 함께 한 일
- {레포명}: {작업 내용} — PR #N / 커밋 N개
- 주요 결정: {결정사항과 맥락}
- 산출물: {생성된 문서, PR, 코드 등}

## ✅ 오늘 완료한 것
- {완료된 이슈/작업 목록}

## 🔄 진행 중 (내일 계속)
- {진행 중인 이슈들}

## 🚧 블로커
- {미해결 블로커}

## 📊 오늘의 숫자
- Linear: N개 완료, N개 진행 중, N개 신규
- 미팅: N개 참석 (총 N시간)
- 커밋: N개
- PR: N개 열림, N개 머지

## 🔍 오늘 발견한 패턴
- {새로 발견된 사용자 패턴/인사이트}
- {User Profile에 업데이트한 내용}
```

피드백이 없으면 "📋 피드백 반영" 섹션은 생략.

### 7. 작성 스타일

`style-guide.md`를 읽고 Bruno 페르소나에 맞게 작성한다.
저녁 인사는 "🌙 Buona sera, 형!"으로 시작.

### 8. 에러 처리

- 아침 Daily 페이지를 못 찾아도 저녁 회고 페이지는 독립적으로 생성
- 특정 도구 실패 시 나머지 데이터로 회고 작성
- 페이지 생성 실패 시 재시도 1회
